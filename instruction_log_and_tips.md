## 💡 2. 指令優化建議 (Prompt Engineering Tips)

為了讓 Agent 能更精準地執行您的意圖，建議採用以下結構來下達指令：

### A. **[動作] + [對象] + [格式/條件] + [後續動作]**

*   **❌ 模糊指令**：
    > "幫我整理這些論文"
*   **✅ 精準指令**：
    > "閱讀 `papersummaries/` 目錄下的所有 `.md` 檔案 **[動作]**，提取其摘要與結論 **[對象]**，整理成一份名為 `Summary_Report.md` 的 Markdown 表格 **[格式]**，完成後推送到 GitHub **[後續動作]**。"

### B. **批次處理的「防呆」機制**

當處理大量檔案時（如您之前處理 20 篇論文），建議加入「標記」或「分批」指令，避免 Agent 超時或混亂：

*   **建議指令**：
    > "處理目錄下的前 5 個檔案。處理完畢後，請將檔名加上 `Review_` 前綴，並刪除原始檔案，以避免重複處理。"

### C. **角色扮演 (Persona)**

您目前的設定非常成功，但可以在特定任務中強化角色：

*   **建議指令**：
    > "你現在是**資深氣象學家**，請用專業但淺顯易懂的語言，解釋 QPE 與 QPF 的差異..."
    > "你現在是**React 前端工程師**，請幫我優化這段代碼的效能..."

### D. **自動化排程 (Cron)**

設定排程時，明確指定「時區」與「頻率」非常重要：

*   **建議指令**：
    > "每天台北時間早上 8 點，搜尋最新的地震學論文，並寄信到 `[Email]`。如果當天沒有新論文，則不發送。"

## 🔄 3. 常用指令模組 (Copy & Paste)

您可以直接複製以下模組並修改括號內容：

### 📝 知識建立模組
```text
請搜尋關於 [主題：如 光纖地震學 DAS] 的最新技術，整理成一份 Markdown 技術筆記。
內容需包含：1. 原理 2. 優缺點 3. 實際案例。
完成後存為 [檔名.md] 並上傳至 GitHub。
```

### 📊 論文批次整理模組
```text
讀取目錄 [目錄路徑] 中的檔案。
針對每個檔案：
1. 提取標題、作者、年份。
2. 撰寫 200 字以內的中文摘要。
3. 加入一段「專業點評」。
將結果寫入新檔案 [Review_原檔名]，並移除舊檔。最後推送到 GitHub。
```

### 🚨 系統監控模組
```text
檢查我的 LLM Token 使用量。如果剩餘額度低於 [30%]，請發送 Email 通知我，並在對話中回報。
```
